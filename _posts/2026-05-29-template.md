## 1. 템플릿(Templates)의 필요성

**[KR]** C++에서 템플릿(Templates)은 자료형(Type)에 구애받지 않고 함수나 클래스를 찍어낼 수 있게 해주는 강력한 기능입니다. 변수의 타입이 없는 파이썬과 달리 C++은 자료형에 매우 엄격하기 때문에, 템플릿을 사용하지 않으면 동일한 로직을 자료형만 바꿔서 계속 복사 및 붙여넣기 해야 하는 비효율이 발생합니다.

예를 들어, 두 숫자 중 더 큰 숫자를 반환하는 `GetMax` 함수를 만든다고 가정해 보겠습니다.
```cpp
// 1. 정수형(int) 비교 함수
int GetMax(int a, int b) {
    return (a > b) ? a : b;
}

// 2. 실수형(float) 비교 함수
float GetMax(float a, float b) {
    return (a > b) ? a : b;
}
```

로직(`return (a > b) ? a : b;`)은 완전히 동일하지만, 매개변수와 반환값의 자료형이 다르다는 이유만으로 함수를 중복해서 작성해야 하는 노가다가 필요합니다.

## 2. 함수 템플릿 (Function Template)

**[KR]** C++ 설계자들은 자료형 자리를 임시 기호로 비워두고 툴(Template)을 만들면, 컴파일러가 알아서 코드를 찍어내도록 만들었습니다. `template <typename T>`를 선언하고, 실제 자료형이 들어갈 자리에 기호 `T`를 적어주면 됩니다.

```cpp
#include <iostream>

// 자료형 자리를 임시 기호 'T'로 비워둔 함수 템플릿
template <typename T>
T GetMax(T a, T b) {
    return (a > b) ? a : b;
}

int main() {
    using namespace std;
    
    // T 자리에 int가 들어가면서 int용 GetMax가 즉석에서 생성됨
    cout << GetMax<int>(10, 20) << endl; 
    
    // T 자리에 float가 들어가면서 float용 GetMax가 즉석에서 생성됨
    cout << GetMax<float>(3.5f, 1.2f) << endl; 
    
    return 0;
}
```

비교 연산자(`>`)만 오버로딩되어 있다면, `MyString`과 같이 사용자가 직접 만든 클래스 객체도 이 템플릿 함수를 통해 비교할 수 있습니다.

## 3. 클래스 템플릿 (Class Template)

**[KR]** 함수뿐만 아니라 클래스도 템플릿으로 만들 수 있습니다. `T* terms;`와 같이 선언하면 `int` 배열도 될 수 있고 `float` 배열도 될 수 있는 유연한 포인터가 됩니다.

```cpp
template <typename T>
class SparsePolynomial {
private:
    size_t length = 0;
    T* terms; // 자료형 T에 따라 결정되는 만능 포인터

public:
    // 매개변수 coef의 자료형도 T에 의해 유동적으로 결정됨
    void NewTerm(T coef, int exp); 
};

int main() {
    // 계수가 int형인 다항식 객체 생성
    SparsePolynomial<int> p_int;
    
    // 계수가 double형인 다항식 객체 생성
    SparsePolynomial<double> p_double;
}
```

## 4. 런타임과 엮어서 이해하는 템플릿의 비밀 (Zero Overhead)

**[KR]** 템플릿을 쓴다고 해서 프로그램이 실행되는 도중(Runtime)에 유동적으로 자료형이 변하는 것이 아닙니다. 템플릿은 프로그램을 빌드하는 **컴파일 타임(Compile time)**에 작동합니다. 

코드에 `GetMax<int>`와 `GetMax<float>`를 적어두면, 컴파일러가 인간을 대신해서 `int`용 함수 코드 하나와 `float`용 함수 코드 하나를 각각 복사 및 붙여넣기하여 기계어 실행 파일에 넣어줍니다. 따라서 프로그램이 실행되는 런타임에는 이미 완벽하게 확정된 `int`형, `float`형 코드로 돌아가기 때문에 속도는 단 1밀리초도 느려지지 않으며 극한의 퍼포먼스를 유지합니다.

우리가 자주 사용하는 C++ 표준 라이브러리의 `std::vector<int>` 등도 바로 이 클래스 템플릿 원리로 만들어진 결과물입니다.
