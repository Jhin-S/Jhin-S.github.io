## 1. Select 및 Pandas와의 비교 학습 (Select & Pandas Comparison)

**[KR]** LeetCode의 SQL 50 트랙 첫 번째 섹션인 'Select' 파트입니다. 가장 기초적인 `SELECT`와 `WHERE` 절의 사용법부터 시작하여, 중복 제거(`DISTINCT`), 열 이름 변경(`AS`), 정렬(`ORDER BY`), 그리고 문자열 길이 측정 함수(`CHAR_LENGTH`) 등을 다룹니다. 특히, 학습 효과를 극대화하기 위해 각 SQL 쿼리를 **Pandas 데이터프레임 조작 코드로 1:1 대응하여 비교(Metacognitive Note-taking)**하는 방식으로 정리했습니다.

**[EN]** This is the first section, 'Select', of LeetCode's SQL 50 track. ## 2. 문제별 핵심 요약 및 코드 비교 (Summary & Code Comparison)

### 2.1. Recyclable and Low Fat Products
**[KR]** `low_fats`가 'Y'이고 `recyclable`이 'Y'인 제품의 ID를 찾는 문제입니다. SQL의 `AND` 연산과 문자열 표기법을 다룹니다. SQL에서 문자열을 표현할 때는 홑따옴표(`'`)를 사용합니다.
**[EN]** The task is to find the IDs of products where `low_fats` is 'Y' and `recyclable` is 'Y'. It covers the `AND` operator and string notation in SQL. In SQL, strings are enclosed in single quotes (`'`).

* **SQL:**
  ```sql
  SELECT product_id
  FROM products
  WHERE low_fats = 'Y' AND recyclable = 'Y';
  ```
 
* **Pandas:**
  ```python
  df[(df['low_fats'] == 'Y') & (df['recyclable'] == 'Y')][['product_id']]
  ```
 

### 2.2. Find Customer Referee
**[KR]** 추천인 ID(`referee_id`)가 2가 아닌 고객의 이름을 찾는 문제입니다. SQL에서 `NULL` 값을 다룰 때는 단순히 `!= 2`만 쓰면 NULL 데이터가 누락되므로, 반드시 `IS NULL` 연산자를 함께 사용해야 한다는 점이 핵심입니다.
**[EN]** The task is to find the names of customers whose `referee_id` is not 2. The key point is that when handling `NULL` values in SQL, using only `!= 2` will miss the NULL data, so you must use the `IS NULL` operator alongside it.

* **SQL:**
  ```sql
  SELECT name
  FROM Customer
  WHERE referee_id != 2 OR referee_id IS NULL;
  ```
 
* **Pandas:**
  ```python
  df[(df['referee_id'] != 2) | (df['referee_id'].isna())][['name']]
  ```
 

### 2.3. Big Countries
**[KR]** 면적(`area`)이 300만 이상이거나 인구(`population`)가 2,500만 이상인 큰 국가의 이름, 인구, 면적을 출력하는 문제입니다. `OR` 조건문의 활용을 다룹니다.
**[EN]** The task is to output the name, population, and area of large countries with an `area` of at least 3 million or a `population` of at least 25 million. It covers the use of `OR` conditions.

* **SQL:**
  ```sql
  SELECT name, population, area
  FROM World
  WHERE area >= 3000000 OR population >= 25000000;
  ```
 
* **Pandas:**
  ```python
  df[(df['area'] >= 3000000) | (df['population'] >= 25000000)][['name', 'population', 'area']]
  ```
 

### 2.4. Article Views I
**[KR]** 자신이 작성한 글을 직접 조회한 적이 있는 저자(Author)의 ID를 오름차순으로 정렬하여 출력하는 문제입니다. 중복을 제거하는 `DISTINCT`, 컬럼 이름을 바꾸는 `AS`, 그리고 오름차순 정렬을 수행하는 `ORDER BY ... ASC`의 조합을 배울 수 있습니다.
**[EN]** The task is to output the IDs of authors who have viewed their own articles, sorted in ascending order. You can learn the combination of `DISTINCT` for deduplication, `AS` for renaming columns, and `ORDER BY ... ASC` for sorting.

* **SQL:**
  ```sql
  SELECT DISTINCT author_id AS id
  FROM Views
  WHERE author_id = viewer_id
  ORDER BY id ASC;
  ```
 
* **Pandas:**
  ```python
  # 1. 조건 필터링 (자기가 자기 글을 본 경우)
  filtered_df = df[df['author_id'] == df['viewer_id']]
  # 2. 필요한 컬럼만 추출하고 이름 변경
  result_df = filtered_df[['author_id']].rename(columns={'author_id': 'id'})
  # 3. 중복 제거 및 정렬
  result = result_df.drop_duplicates().sort_values(by='id', ascending=True)
  ```
 

### 2.5. Invalid Tweets
**[KR]** 트윗 내용(`content`)의 길이가 15자를 초과하는 유효하지 않은 트윗의 ID를 찾는 문제입니다. SQL의 문자열 길이 측정 함수인 `CHAR_LENGTH()`를 학습합니다.
**[EN]** The task is to find the IDs of invalid tweets where the length of the `content` strictly exceeds 15 characters. It covers `CHAR_LENGTH()`, a string length measurement function in SQL.

* **SQL:**
  ```sql
  SELECT tweet_id
  FROM Tweets
  WHERE CHAR_LENGTH(content) > 15;
  ```
 
* **Pandas:**
  ```python
  # .str 을 붙여야 문자열 데이터로 인식하고 .len()으로 길이를 잴 수 있습니다.
  # (len(df['content'])는 데이터프레임의 총 행 개수를 반환하므로 주의!)
  df[df['content'].str.len() > 15][['tweet_id']]
  ```
